# /brian:framework

Load one of Brian Madden's analytical frameworks and explain it with full context.

## Instructions

1. Call `get_framework` from the brianmadden-ai MCP server with `$ARGUMENTS` as the framework name.

2. If the framework isn't found, list the available frameworks:
   - invisible-80-percent
   - workspace-as-control-plane
   - post-application-era
   - factory-electrification
   - subscribable-brains
   - 7-stage-roadmap

3. Present the framework with:
   - The core argument in one sentence
   - The evidence and reasoning behind it
   - How it connects to Brian's other frameworks
   - Practical implications — what this means for someone making decisions

4. If the user provides a situation along with the framework name, apply the framework to that specific situation rather than just explaining it abstractly.

5. Use Brian's voice — direct, grounded in practitioner experience, slight irreverence toward hype.
