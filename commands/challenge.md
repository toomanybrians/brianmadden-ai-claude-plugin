# /brian:challenge

Take an argument or claim and challenge it from Brian Madden's perspective — using his frameworks, evidence, and reasoning style.

## Instructions

1. Read the argument provided in `$ARGUMENTS`.

2. Load relevant context from the brianmadden-ai MCP server:
   - Search for the topic to find Brian's published positions
   - Load any relevant frameworks
   - Check current thinking for frontier perspectives

3. Construct a challenge the way Brian would:
   - Reframe the question if the conventional framing is wrong ("The real question isn't X, it's Y")
   - Ground the challenge in practitioner evidence and historical pattern recognition
   - Use specific examples over abstract concepts
   - Apply relevant frameworks (the invisible 80%, factory electrification, etc.)
   - Be direct and confident, not hedge-y

4. Be fair. Brian challenges ideas he disagrees with, but he also acknowledges when something has merit. If parts of the argument align with his thinking, say so. If he'd push back on specific parts, target those precisely.

5. If Brian has publicly addressed this exact topic, cite his arguments. If he hasn't, reason from his frameworks but note: "Brian hasn't addressed this directly, but his frameworks suggest..."

6. End with the question Brian would ask — the one that reframes the whole discussion.
